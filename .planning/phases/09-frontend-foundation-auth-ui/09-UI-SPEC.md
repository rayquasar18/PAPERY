## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label spacing |
| sm | 8px | Compact element spacing, form field internal padding |
| md | 16px | Default element spacing, card padding, button padding-x |
| lg | 24px | Section padding, form group spacing, sidebar item padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, auth page vertical spacing |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- **44px** minimum touch target height for mobile interactive elements (buttons, sidebar items, form inputs)
- **280px** sidebar expanded width; **56px** sidebar collapsed (icon-only) width
- **320px** minimum chat panel width; **480px** default chat panel width
- **12px** border radius for cards (shadcn `--radius` base = 0.75rem)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (0.875rem) | Regular (400) | 1.5 (21px) | Default text, form labels, descriptions, table cells |
| Label | 12px (0.75rem) | Regular (400) | 1.5 (18px) | Form field labels, badges, metadata, timestamps |
| Heading | 20px (1.25rem) | Semibold (600) | 1.3 (26px) | Page titles, section headers, card titles |
| Display | 30px (1.875rem) | Semibold (600) | 1.2 (36px) | Auth page branding headline, hero text |

**Weights used:** Regular (400), Semibold (600) — 2 weights total.

> Note: shadcn/ui defaults to 14px body. Inter is optimized for screen readability at 14-16px. 14px chosen for information-dense SaaS layout. Two-weight system (Regular + Semibold) maintains clear visual hierarchy while reducing font payload and cognitive complexity.

**Font stack:**
```css
--font-sans: 'Inter', ui-sans-serif, system-ui, -apple-system, sans-serif;
```

### 60/30/10 Rule Enforcement

- **60% Dominant:** Background surfaces (`--background`) — page body, empty areas
- **30% Secondary:** Cards, sidebar, nav, modals (`--card`, `--sidebar`, `--muted`)
- **10% Accent:** Reserved for ONLY these specific elements:
  - Primary action buttons (CTA buttons)
  - Active sidebar nav item indicator (left border or background)
  - Active tab indicator
  - Focus rings on form inputs
  - Links in body text
  - Loading spinner
  - Toggle switches (active state)
  - Auth page branding gradient accent

> Accent is NEVER used for: body text, borders, disabled states, section backgrounds, or decorative elements.

### Focal Point Declaration

| Page | Primary Focal Point | Rationale |
|------|-------------------|-----------|
| Dashboard (empty state) | Empty state illustration + "Create project" CTA button (centered in main content area) | First-time users must immediately see what to do next — the CTA is the only accent-colored element on an otherwise neutral page, creating a clear visual anchor |
| Dashboard (with data) | Project cards grid (top-left of main content area) | Users return to check on their projects — the first card in reading order serves as the entry point |
| Auth pages (login/register) | Form panel's primary CTA button ("Sign in" / "Create account") | The form is the sole task — the CTA button draws the eye as the completion target |

## Interaction Patterns

### Loading States

| Scenario | Pattern |
|----------|---------|
| Auth form submission | Button shows spinner + disabled state, form fields disabled |
| Page navigation | Next.js loading.tsx with skeleton UI |
| Dashboard initial load | Skeleton cards (3 placeholders) |
| Sidebar loading | Skeleton items (3 lines) |
| API call in progress | TanStack Query `isPending` drives component loading state |

### Focus Management

| Scenario | Behavior |
|----------|----------|
| Auth page load | Auto-focus first empty field (email on login, name on register) |
| Form submission error | Focus first field with error |
| Modal/sheet open | Focus first interactive element inside |
| Modal/sheet close | Return focus to trigger element |
| Tab key in sidebar | Sequential through visible items only |

### Keyboard Shortcuts (v1)

| Shortcut | Action |
|----------|--------|
| `Cmd/Ctrl + K` | Open global search (placeholder) |
| `Cmd/Ctrl + B` | Toggle sidebar |
| `Cmd/Ctrl + J` | Toggle chat panel |
| `Escape` | Close modal/sheet/dropdown |

### Animation Contracts

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Sidebar collapse/expand | Width transition | 200ms | ease-in-out |
| Chat panel open/close | Slide from right + width transition | 200ms | ease-in-out |
| Mobile sidebar sheet | Slide from left | 200ms | ease-out (open) / ease-in (close) |
| Button hover | Background color | 150ms | ease |
| Toast enter | Slide up + fade in | 200ms | ease-out |
| Toast exit | Slide right + fade out | 150ms | ease-in |
| Skeleton pulse | Opacity pulse | 1500ms | ease-in-out (infinite) |
| Page transition | None — instant (Next.js App Router native) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| WCAG level | 2.1 AA minimum |
| Color contrast | 4.5:1 for body text, 3:1 for large text (AA) |
| Focus indicators | 2px ring using `--ring` color, 2px offset |
| Screen reader | All interactive elements have accessible labels |
| Form errors | `aria-invalid`, `aria-describedby` linking error message |
| Live regions | Toast notifications use `role="status"` with `aria-live="polite"` |
| Skip link | "Skip to main content" link, visible on focus, first focusable element |
| Reduced motion | Respect `prefers-reduced-motion: reduce` — disable all transitions |
| Language | `<html lang>` attribute matches current locale |
