<title>Configure next-intl routing and request</title>
<read_first>
- .planning/phases/09-frontend-foundation-auth-ui/09-RESEARCH-i18n.md (routing.ts, request.ts, navigation.ts, global.d.ts ‚Äî exact code)

<action>
1. Create `frontend/src/i18n/routing.ts`:
   ```typescript
   import { defineRouting } from 'next-intl/routing';

   export const routing = defineRouting({
     locales: ['en', 'vi'],
     defaultLocale: 'en',
     localePrefix: 'always',
     localeCookie: {
       name: 'NEXT_LOCALE',
       maxAge: 60 * 60 * 24 * 365,
     },
   });
   ```
2. Create `frontend/src/i18n/request.ts`:
   ```typescript
   import { getRequestConfig } from 'next-intl/server';
   import { hasLocale } from 'next-intl';
   import { routing } from './routing';

   export default getRequestConfig(async ({ requestLocale }) => {
     const requested = await requestLocale;
     const locale = hasLocale(routing.locales, requested)
       ? requested
       : routing.defaultLocale;

     return {
       locale,
       messages: (await import(`../locale/${locale}.json`)).default,
     };
   });
   ```
3. Create `frontend/src/i18n/navigation.ts`:
   ```typescript
   import { createNavigation } from 'next-intl/navigation';
   import { routing } from './routing';

   export const { Link, redirect, usePathname, useRouter, getPathname } =
     createNavigation(routing);
   ```
4. Create `frontend/src/i18n/global.d.ts` for type-safe translations using `en.json` as source of truth.
</action>

## Threat Model

<threat_model>
| Threat | Severity | Likelihood | Mitigation |
|--------|----------|------------|------------|
| FOUC on theme switch | LOW | MEDIUM | next-themes script injection prevents flash; `disableTransitionOnChange` |
| Locale cookie manipulation | LOW | LOW | Cookie only affects display locale, no security impact |
| SSR hydration mismatch | MEDIUM | MEDIUM | `suppressHydrationWarning` on html; useEffect for client-only state |
</threat_model>
