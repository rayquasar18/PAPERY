```dockerfile
# ============================================================================
# PAPERY Backend — Production Dockerfile
# Multi-stage build: builder installs deps, runtime copies only .venv
# ============================================================================

# --- Stage 1: Builder — install dependencies with uv ---
FROM python:3.12-slim AS builder

WORKDIR /app

# Install uv package manager
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

# Install only production dependencies (no dev group)
COPY backend/pyproject.toml backend/uv.lock ./
RUN uv sync --locked --no-dev --no-install-project

# --- Stage 2: Runtime — minimal production image ---
FROM python:3.12-slim AS runtime

# Security: run as non-root user
RUN groupadd --gid 1000 papery && \
    useradd --uid 1000 --gid papery --shell /bin/bash --create-home papery

WORKDIR /app

# Copy only the virtual environment from builder
COPY --from=builder /app/.venv /app/.venv

# Copy application source code
COPY backend/ /app/

# Set PATH to use .venv binaries
ENV PATH="/app/.venv/bin:$PATH" \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    WEB_CONCURRENCY=2

# Switch to non-root user
USER papery

EXPOSE 8000

# Health check
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
    CMD ["python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/api/v1/health')"]

# Production entrypoint: gunicorn with uvicorn workers
CMD ["gunicorn", "app.main:app", \
     "--bind", "0.0.0.0:8000", \
     "--workers", "${WEB_CONCURRENCY:-2}", \
     "--worker-class", "uvicorn_worker.UvicornWorker", \
     "--timeout", "120", \
     "--graceful-timeout", "30", \
     "--keep-alive", "5", \
     "--log-level", "info", \
     "--access-logfile", "-"]
```

Key design points:
- Stage 1 (builder): installs deps with `uv sync --locked --no-dev` — no test/dev tools in production
- Stage 2 (runtime): fresh `python:3.12-slim`, copies only `.venv` — minimal image size
- Non-root `papery` user (security best practice)
- `PYTHONUNBUFFERED=1` for proper log streaming in containers
- `WEB_CONCURRENCY=2` default, overridable via env var
- `HEALTHCHECK` uses stdlib `urllib` to hit `/api/v1/health` (no curl needed)
- `gunicorn` with `uvicorn_worker.UvicornWorker` (not deprecated `uvicorn.workers`)
- No `--reload` flag (production)
- `--access-logfile -` sends access logs to stdout for Docker log collection

**Note on CMD with env var interpolation:** Docker CMD exec form doesn't expand env vars. Use shell form or entrypoint script. Since we need `$WEB_CONCURRENCY`, use shell form:

Actually, to properly support env var expansion for `--workers`, use shell form CMD:

```dockerfile
CMD gunicorn app.main:app \
    --bind 0.0.0.0:8000 \
    --workers ${WEB_CONCURRENCY:-2} \
    --worker-class uvicorn_worker.UvicornWorker \
    --timeout 120 \
    --graceful-timeout 30 \
    --keep-alive 5 \
    --log-level info \
    --access-logfile -
```

(Shell form, not JSON exec form, so `$WEB_CONCURRENCY` gets expanded by `/bin/sh -c`)
