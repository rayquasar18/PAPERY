# Quick Task Plan: Refactor AuthService to Class-Based with DI

**ID:** 260410-u0m
**Type:** refactor
**Created:** 2026-04-10

---

## Goal

Refactor `auth_service.py` from standalone functions to a class-based `AuthService` with dependency injection. This establishes the canonical pattern for all future services (ProjectService, DocumentService, ChatService, etc.) — constructor receives `AsyncSession`, creates repo once, all methods are instance methods.

---

## Analysis

### Current State
- `auth_service.py` has 8 standalone async functions
- 6 of them create `UserRepository(db)` independently (duplicated 6x)
- `logout_user` and `send_verification_email` don't need `db` (no repo creation)
- Router (`auth.py`) imports module `from app.services import auth_service` and calls `auth_service.function(db, ...)`
- Router `refresh` endpoint and `resend_verification` endpoint directly create `UserRepository(db)` — this logic should also move into the service
- `scripts/create_first_superuser.py` imports `from app.services.auth_service import create_first_superuser`

### Impact Surface
| File | Change Type |
|------|-------------|
| `backend/app/services/auth_service.py` | Major — wrap all functions into `AuthService` class |
| `backend/app/api/v1/auth.py` | Moderate — instantiate `AuthService(db)`, fix all calls, move UserRepository usage from router into service |
| `backend/app/services/__init__.py` | Minor — update exports |
| `backend/scripts/create_first_superuser.py` | Minor — update import path |

## Tasks

### Task 1: Refactor `auth_service.py` to class-based `AuthService`

**File:** `backend/app/services/auth_service.py`

**Changes:**
1. Create `AuthService` class with `__init__(self, db: AsyncSession)`
   - `self._db: AsyncSession` — stored for methods that need raw session
   - `self._user_repo: UserRepository` — created once in constructor
2. Convert all 8 standalone functions to instance methods:
   - `register_user(self, email, password)` — remove `db` param, use `self._user_repo`
   - `authenticate_user(self, email, password)` — remove `db` param, use `self._user_repo`
   - `logout_user(self, ...)` — static-ish, no repo needed, but keep as instance method for consistency
   - `rotate_refresh_token(self, old_payload)` — remove `db` param, use `self._user_repo`
   - `verify_email(self, token)` — remove `db` param, use `self._user_repo`
   - `send_verification_email(self, email, user_uuid)` — no change needed (no db), keep as instance method
   - `request_password_reset(self, email)` — remove `db` param, use `self._user_repo`
   - `reset_password(self, token, new_password)` — remove `db` param, use `self._user_repo`
   - `create_first_superuser(self)` — remove `db` param, use `self._user_repo`
3. Add new helper method for router to avoid router creating UserRepository directly:
   - `get_user_by_uuid(self, user_uuid)` — used by refresh endpoint
   - `get_user_by_email(self, email)` — used by resend-verification endpoint
4. Keep backward-compatible module-level factory (optional, for `create_first_superuser` script):
   - Top-level `create_first_superuser(db)` that wraps `AuthService(db).create_first_superuser()`

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Class instance per request | `AuthService(db)` in each endpoint | Matches request lifecycle; no singleton state leaks between requests |
| `_user_repo` private attribute | Underscore prefix | Internal implementation detail, not part of public API |
| `logout_user` as instance method | Keep as instance method even though it doesn't use repo | Consistency — all auth operations go through AuthService |
| Module-level `create_first_superuser` wrapper | Keep backward compat | Script uses direct import; minimal change for ergonomics |
| Router UserRepository → service methods | Move into service | Router should ONLY call service layer, never repositories directly |

---

## Constraints

- **No functional changes** — purely structural refactor
- **No new dependencies** — only internal restructuring
- **Backward-compatible** — `create_first_superuser` script still works
- **Pattern-setting** — this becomes the template for `ProjectService`, `DocumentService`, etc.
